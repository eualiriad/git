Entendendo Comandos Git

git add: O comando git add é usado para adicionar as alterações feitas nos arquivos do repositório Git à área de preparação (também conhecida como "staging area" ou "stage) para que possam ser incluídas no próximo commit. O comando git add pode ser usado para adicionar novos arquivos, atualizar arquivos existentes ou remover arquivos do repositório. git commit: O comando git commit é usado para confirmar as alterações feitas nos arquivos do repositório Git que foram adicionados à área de preparação com o comando git add. O comando git commit cria um novo versionamento das alterações e adiciona uma mensagem de commit que descreve as alterações feitas. Cada commit no Git tem um ID exclusivo e uma mensagem que o descreve. git status: O comando git status é um comando utilizado no Git para exibir o estado atual do repositório. Ele mostra quais arquivos foram modificados, quais arquivos estão na área de preparação (ou "staging area") prontos para serem confirmados (com o comando "git commit"), e quais arquivos não foram rastreados pelo Git.
